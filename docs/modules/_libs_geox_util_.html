SIGFIG_HASH_LENGTH

SIGFIG_HASH_LENGTH: number[] = [0, 5, 7, 8, 11, 12, 13, 15, 16, 17, 18]
  • Defined in libs/geox/util.ts:94

Significant Figure Hash Length

This is a quick and dirty lookup to figure out how long our hash should be in order to guarantee a certain amount of trailing significant figures. This was calculated by determining the error: 45/2^(n-1) where n is the number of bits for a latitude or longitude. Key is # of desired sig figs, value is minimum length of the geohash.
